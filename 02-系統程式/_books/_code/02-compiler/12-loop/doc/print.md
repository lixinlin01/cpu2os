
### 原理講解 (它是如何辦到的？)

當 Parser 讀到 `print("fact(5)=", fact(5))` 時：
1. 它發現了字串 `"fact(5)="`。因為這不能拿來加減乘除，它會把這段字串偷偷塞進一個備份空間 `string_pool` (索引為 `0`)。然後產生指令：`PRINT_STR 0`。
2. 接著遇到逗號 `,`，它繼續解析後面的 `fact(5)`。這會觸發函數呼叫的中間碼生成，最後把算出來的暫存變數 `t10` 交出來，產生指令：`PRINT_VAL t10`。
3. 等 `print(...)` 括號結束後，產生最後一個指令：`PRINT_NL`。

當 VM 執行這段指令時，它不會立刻印出來，而是先把它們塞進**緩衝區 (`self.print_buf`)**：
* 遇到 `PRINT_STR 0`：將字串池第0號 `"fact(5)="` 放進緩衝區。
* 遇到 `PRINT_VAL t10`：將記憶體中的數值 `120` 放進緩衝區。
* 遇到 `PRINT_NL`：一口氣將緩衝區內的文字 `join` 起來（中間自動補上空格）印到畫面上。這就完美還原了 Python 原生 print 多個參數的行為了！
